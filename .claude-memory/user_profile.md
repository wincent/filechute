Greg Hurrell is an experienced developer building Filechute as a solo
project with Claude assistance. Key traits relevant to collaboration:

- Deep familiarity with macOS, SwiftUI, and AppKit bridging patterns.
  Explanations can use technical terminology without hand-holding.
- Consistently targets native macOS behavior (e.g. "matching Finder's
  behavior" appears across many feature requests and bug reports).
- Iterates on visual details through multiple feedback rounds — expects
  pixel-level accuracy on UI elements (button sizing, highlight
  styling, layout spacing).
- Provides precise bug reports: specifies which columns are slow vs
  fast, which key combos work vs don't, with screenshots.
- For major features, sometimes asks for an ADR (in docs/adrs/) before
  implementation to align on design.
